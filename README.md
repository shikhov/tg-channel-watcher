# Telegram Channel Watcher
Приложение отслеживает новые сообщения в каналах/чатах Telegram и пересылает те, которые подпадают под заданные правила. Пример работы: https://t.me/travelekb

## Реализация
Приложение написано на Python 3.10 с использованием фреймворка Telethon. Хранение данных в MongoDB. Для развертывания прилагается Dockerfile.

## Развертывание

- Заполнить `config.py`:
  - `DBNAME` — имя БД
  - `CONNSTRING` — строка соединения с MongoDB
  
  *(также эти параметры можно передать через переменные среды)*
- Получить api_id и api_hash для приложения на https://my.telegram.org
- С помощью фреймворка Telethon произвести аутентификацию на сервере Telegram, сохранить string session. См.: https://docs.telethon.dev/en/stable/concepts/sessions.html#string-sessions
- Создать БД с именем, указанным в `config.py`, в ней создать коллекции settings и profiles
- В коллекции settings создать документ с _id: "settings" и заполнить его:
    - `sleeptimer` — период опроса каналов в секундах
    - `api_id`, `api_hash`, `session` — значения, полученные на предыдущих шагах
    - `logchatid` — ID чата, куда будут сбрасываться ошибки. Опциональное поле
    - `debug` — расширенный вывод сообщений в лог. Опциональное поле, по умолчанию False

Пример:
```json
"_id": "settings",
"sleeptimer": 300,
"logchatid": -1002243669517,
"api_id": 123456,
"api_hash": "your_api_hash",
"session": "your_string_session",
"debug": true
```

- В коллекции profiles создать и заполнить документы профилей:
  - `name` — имя профиля
  - `enable` — активен/неактивен
  - `channels` — список идентификаторов Telegram-каналов (чатов)
  - `output` — список объектов output. Объект output состоит из:
    - `rules` — правила фильтрации сообщений. В правиле может быть:
      - одно поле `regex` — регулярное выражение, на которое проверяется соообщение. Пример:
        ```json
        {
          "regex" : "iphone (16|17) pro(\\s+max)?"
        }
        ```
      - поля `regex` и `eval` — в случае совпадения с регулярным выражением из поля `regex` дополнительно выполняется python-код из поля `eval`. Если результат кода True, правило считается сработавшим. Код выполняется для всех найденных совпадений. Для использования фрагментов из скобок регулярного выражения можно использовать переменную `m`. Пример:
        ```json
        {
          "regex" : "(\\$\\s?\\d+|\\d+\\s?\\$)",
          "eval" : "int(m.group(1).replace('$', '')) <= 800"
        }
        ``` 
      - одно поле `eval` — для проверки правила выполняется только python-код. Примеры использования объекта `msg: Message` фреймворка Telethon:
        ```json
        {        
          "eval" : "msg.voice or msg.reply_markup"
        },
        {        
          "eval" : "msg.file and msg.file.name and '.txt' in msg.file.name.lower()"
        }
        ``` 

        
    - `any_matching` — true, если достаточно срабатывания одного из правил; false, если сработать должны все
    - `hide_forward` — опциональное поле. false (по умолчанию) - сообщение пересылается обычным форвардом (видно, из какого оно канала); true - сообщение пересылается без отметки о форварде. Опция применяется только к источникам-каналам (не к чатам). Также опция не работает в режиме `all_messages`, сообщения в этом случае всегда пересылаются форвардом.
    - `filter_dupes` — фильтровать повторяющиеся сообщение (с полностью совпадающим текстом). Не применяется в режиме `all_messages`. Опциональное поле, по умолчанию false.
    - `all_messages` — опциональное поле. Если выставлено в true, пересылаются все сообщения. Поля `rules` и `any_matching` в этом случае можно опустить. Для исключения сообщений из пересылки можно использовать поле `ex_rules`
    - `ex_rules` — список правил исключения. Работают только для режима all_messages. Состав аналогичен полю `rules`. При срабатывании любого правила сообщение **не пересылается**
    - `output_channel` — идентификатор целевого канала, куда будут пересылаться сообщения


    _Идентификатор канала (или чата) может быть буквенным именем, ссылкой на вступление или числовым ID (см. документацию к Telethon)_

Пример профиля с rules:
```json
"name": "travelekb",
"enable": true,
"channels": [
    "piratesru",
    "vandroukiru",
    "turs_sale"
],
"output": [
  {
    "rules": [
      {
        "regex" : "екатеринбург|ебург|екб"
      }
    ],
    "any_matching": true,
    "hide_forward": true,
    "filter_dupes": true,
    "output_channel": "travelekb"
  }
]
```

Пример профиля с all_messages:
```json
"name": "test_all",
"enable": true,
"channels": [
  "durov"
],
"output": [
  {
    "all_messages": true,
    "output_channel": "t.me/+_alVeiHBjGU7MGFi"
  },
  {
    "all_messages": true,
    "output_channel": 68651304,
    "ex_rules": [
      {
        "regex": "my new photo|\\bsperm\\b"
      },
      {
        "eval": "msg.sticker"
      }
    ]
  }
]
```

- Развернуть Dockerfile

## Ограничения
В режиме `rules` не пересылаются альбомы — только первое сообщение из альбома, если на нем сработают правила.
В режиме `all_messages` альбомы пересылаются, но отфильтровать их можно только по первому сообщению.

Если у сообщения выставлен флаг, запрещающий пересылку, оно, тем не менее, будет переслано, но не форвардом (это невозможно), а копией.

Также нужно помнить множество нюансов работы с Telegram, описание которых выходит за рамки этого мануала (например, учетки без премиума не могут отправлять сообщения с премиум-фишками, но форвардить могут и т.д.).
